## Chunk 4: Code Quality Fixes from Review

### Task 5: Fix NAT Gateway detection in detectService, then clean pricingspec.go

**Context:** The `pricingspec.go` switch has hardcoded `"nat_gateway"` and `"nat-gateway"` strings
because `detectService()` does NOT handle these plain-string inputs. When a user passes
`resource_type: "nat_gateway"` (no `aws:` prefix):

1. `normalizeResourceType("nat_gateway")` → returns `"nat_gateway"` (no `aws:` prefix, just lowercased)
2. `detectService("nat_gateway")` → no fast-path match (serviceNATGW is `"natgw"`), no
   `strings.Contains` match → returns `"nat_gateway"` unchanged
3. Only the hardcoded strings in `pricingspec.go` catch it

The proper fix is to add these aliases to `detectService`'s fast path, THEN remove them from the switch.

**Files:**
- Modify: `internal/plugin/projected.go:1136-1151` (detectService fast path)
- Modify: `internal/plugin/pricingspec.go:59`
- Test: `internal/plugin/service_cache_test.go`

- [ ] **Step 1: Write test for NAT Gateway alias detection**

Add test cases to `TestServiceResolver_AllSupportedServices` in
`internal/plugin/service_cache_test.go`. The existing test uses a 2-field struct
`{resourceType, expectedService}` with `resourceType` as the sub-test name (line 391).

Add these entries after the existing `{"aws:ec2/natGateway:NatGateway", "natgw"}` line (387):

```go
// NAT Gateway plain-string aliases (must resolve via detectService fast path)
{"nat_gateway", "natgw"},
{"nat-gateway", "natgw"},
{"natgateway", "natgw"},
```

- [ ] **Step 2: Run test to verify it fails**

Run: `go test ./internal/plugin/... -run TestServiceResolver_AllSupportedServices -v -count=1`
Expected: FAIL — `"nat_gateway"` currently resolves to `"nat_gateway"`, not `"natgw"`

- [ ] **Step 3: Add NAT Gateway aliases to detectService fast path**

